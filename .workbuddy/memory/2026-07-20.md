## 修复：🔊 发音没声音 + audio_url 为空

**现象**：用户点播放没声音，且发现 `words.audio_url` 在数据库是空。

**根因**：
- `audio_url` 是按设计有意留空的（`schema.sql` 注释"可空，前端可用语音合成"）；前端原 `speak()` 只用 Web Speech API，从不读该字段。
- 没声音的真凶是 `speechSynthesis` 依赖本机装了英文 TTS 语音，很多 Windows 没装→静默失败；外加 Chrome `cancel()` 紧跟 `speak()` 首调被吞的 bug。

**修复**：
- `scripts/build-word-data.js`：每个单词按 `https://dict.youdao.com/dictvoice?audio={word}&type=2` 拼出真实音频地址写入 `audio_url`（type=2 美音，可改 1 英音）。24,680 词 100% 覆盖。
- `public/js/app.js`：`speak(word, audioUrl)` 改为优先 `<audio>` 播放真实音频，失败回退 Web Speech（修了首调延迟 + cancel 竞态）。
- `db/seed.js` / `db/setup.js`：INSERT 增加 `audio_url` 列，用 `ON DUPLICATE KEY UPDATE audio_url=VALUES(audio_url)`，使重跑 `db:seed` 能把**已有行的 audio_url 补上**（无需删库）。

**用户操作**：重跑 `npm run db:seed`（已存在的单词会补 audio_url）；刷新浏览器即可听到有道 TTS 发音。沙箱网络挡了有道端点（返回空体），但用户本机（国内）正常。

## 关键排查：各词库已学=0 实为"旧服务未重启"

**现象**：整体"已学单词=3"，但各词库已学都显示 0。

**真正根因**：用户在 3000 端口跑的服务是**旧后端代码**（之前 `node src/server.js` 报 `EADDRINUSE` 即证明端口已被占）。旧 `/api/libraries` 不含 `learned_count` 字段 → 前端读到 `undefined` → 各词库显示 0；旧服务也无 `/api/words/search` 路由（返回 `Cannot GET`）。整体 3 来自早就有的 `/review/stats`。

**本次改动**：
